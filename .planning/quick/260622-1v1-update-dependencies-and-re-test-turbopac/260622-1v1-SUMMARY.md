## Summary

Updated Next.js to 16.2.9, React to 19.2.7, and compatible same-major application and development dependencies. Kept pnpm as the package manager and deferred major migrations.

Added a Turbopack tracing exclusion to the configurable DuckDB path. This removed the build warning that the whole project was being traced through `src/lib/db.ts`.

### Memory measurements

- Next 16.1.6 cold `/` baseline: approximately 2.69 GB sampled RSS.
- Next 16.2.9 cold `/`: approximately 2.80 GB peak RSS, falling after compilation.
- Cold request: 3.6 seconds; subsequent requests: 18–24 ms.
- No monotonic growth was observed from repeated root requests; the large allocation is a cold Turbopack compilation spike rather than retained page data.

### Verification

- `pnpm build`: Turbopack compilation succeeds in 7.2 seconds without the whole-project tracing warning; build stops at a pre-existing TypeScript error in `src/app/api/adaptive/bursts/route.ts`.
- `pnpm typecheck`: generated Next types are clean; existing application type errors remain.
- `pnpm test --run`: 486 passed, 8 existing failures.
- `pnpm lint`: 71 existing errors and 98 warnings.
