## Summary

Kept the summary card and added passive visual overlays so hotspot movement is derivable from the map and cube rather than stated outright.

### What changed

1. **Map trajectory overlay**
   - Added `MapHotspotTrajectoryLayer` to draw STKDE hotspot paths on the demo map
   - Renders line strings plus node markers from `stkdeResponse.sliceResults`
   - Uses the same tracked hotspot snapshots as the summary card

2. **3D trajectory overlay**
   - Added `HotspotTrajectoryOverlay` to draw projected hotspot paths inside the cube
   - Aligns marker positions with the slice stack Y ordering and projected lon/lat coordinates
   - Hidden in focus mode to avoid cluttering single-slice inspection

3. **Shared evolution data**
   - Reused the existing hotspot evolution utility and hook
   - The explicit status card remains, but the visual answer now comes from the scene itself
