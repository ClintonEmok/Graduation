Current issue:
- useCrimeData calculates buffered range client-side (lines 104-109): bufferedStart = start - bufferDays, bufferedEnd = end + bufferDays
- It passes already-buffered epoch values to API
- API ALSO calculates buffering with buildBufferedRange (line 128)
- Result: double buffering when bufferDays > 0

Solution:
- Remove client-side buffering from useCrimeData
- Always pass visible startEpoch/endEpoch to API
- API applies bufferDays (default 30) to visible range
</context>
