In `src/components/viz/DataPoints.tsx`, modify the `onBeforeCompile` callback (line ~499):

```typescript
const onBeforeCompile = (shader: any) => {
  if (meshRef.current) {
    const material = meshRef.current.material as THREE.Material;
    material.userData.shader = shader;
    // Stable cache key — prevents WebGL recompilation across instances
    material.customProgramCacheKey = () => 'ghosting-v3-stable';
  }

  applyGhostingShader(shader, {
    useColumns: Boolean(columns),
    typeMapSize: 36,  // now hardcoded — matches shader source
    districtMapSize: 36,
  });
};
```

**Step 2: Compute bounding spheres after instance updates**

In DataPoints.tsx, after the useLayoutEffect that sets instance matrices (line ~212-239), add explicit bounding sphere computation:

After `meshRef.current.instanceMatrix.needsUpdate = true;` (line 238), add:
```typescript
// Compute bounding sphere for frustum culling after all instances are placed
meshRef.current.computeBoundingSphere();
```

This ensures the InstancedMesh's bounding sphere covers ALL instances, not just the first one. Without this, Three.js may compute a bounding sphere that doesn't cover distant instances, causing premature culling.

Do the same in AggregatedBars.tsx after its matrix update.

**Step 4: Add LOD for DataPoints (8.5M instances)**

The existing shader already has `uLodFactor` uniform and uses checkerboard dither + point shrinkage in the vertex shader. Wire the existing `useAggregationStore.lodFactor` to actual camera distance:

In the useFrame (same one updated in Task 1), compute LOD from camera distance:
```typescript
// In useFrame, after reading lodFactor from store:
const cameraDistance = state.camera.position.length();
// Map distance to LOD: 0 (close) → 1 (very far, max culling)
// Camera starts at ~225 units, so normalize 80-300 range
const distanceLod = Math.max(0, Math.min(1, (cameraDistance - 80) / 220));
// Blend store-driven LOD with distance-driven LOD
const effectiveLod = Math.max(lodFactor, distanceLod);
material.userData.shader.uniforms.uLodFactor.value = effectiveLod;
```

The existing shader code already handles `uLodFactor`:
- Vertex shader: shrinks point geometry (`transformed * (1.0 - uLodFactor)`)
- Fragment shader: checkerboard dither discards at high LOD values
