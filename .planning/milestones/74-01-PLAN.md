# Plan 74-01: Coordination Polish

## Goal

Make map, 3D, and timeline stay synchronized with the active slice so that apply, playback, and navigation feel coherent across all views.

## Requirement

**FLOW-10**: Map, cube, and timeline stay synchronized with the active slice while chrome stays minimal.

## Success criteria (what must be TRUE after this plan)

1. Applying a slice lands the user with the correct slice active in inspect/timeline/map
2. Map shows crime data scoped to the active slice's time range
3. Playback cycles through slices in all views simultaneously (timeline highlight follows)
4. Scrubbing the SliceScrubber in InspectPanel updates both 3D view and timeline highlight
5. The timeline's active slice highlight stays in sync with inspect panel's active index

## Background / Coordination Gap Analysis

Research revealed 8 gaps across the coordination system. The root cause is that two different "active slice" indicators exist with no bridge between them:

| Store | Field | Used by |
|-------|-------|---------|
| `useDashboardDemoCoordinationStore` | `activeSliceIndex` (number) | Demo3dSpatialView, DemoInspectPanel (playback/scrub), SliceScrubber |
| `useSliceDomainStore` | `activeSliceId` (UUID string) | DemoDualTimeline (via `selectActiveSliceId`) |

Additional gaps:
- Map (`MapVisualization`) reads only `selectedIndex` — ignores `activeSliceIndex` entirely
- No `activeSliceIndex` is set immediately on apply (relies on async 3D fetch to set it)
- 3D view overrides `activeSliceIndex` to last slice on every data refetch
- `activeSliceIndex` changes bypass the reconciliation system

## Tasks

### Task 1: Bridge `activeSliceIndex` → `activeSliceId` for timeline sync

**Why:** The timeline highlights based on `activeSliceId` (domain store), but playback/scrubbing in InspectPanel changes `activeSliceIndex` (coordination store). Without a bridge, the timeline never updates during playback or scrubbing.

**What:**
- In `DemoInspectPanel.tsx`, add a `useEffect` that watches `activeIndex` and `visibleSlices`
- When `activeIndex` changes and `visibleSlices[activeIndex]` exists, call `setActiveSliceId` on the domain store with `visibleSlices[activeIndex].sourceSliceId`
- This ensures every index change propagates to the domain store, which updates the timeline highlight

**Files:** `src/components/dashboard-demo/DemoInspectPanel.tsx`

### Task 2: Set `activeSliceIndex` immediately on apply

**Why:** When a user clicks "Apply" on a draft, `addSliceFromBin` sets `activeSliceId` on the domain store, but `activeSliceIndex` on the coordination store stays at its previous value (default 0). The inspect panel then shows slice[0] instead of the newly applied slice.

**What:**
- In `handleApplySingleDraft` (DemoSlicePanel.tsx), after `applySingleGeneratedBin` returns true, calculate the new slice's index in the domain store's sorted visible range slices
- Call `setActiveSliceIndex(index)` to set it immediately, bypassing the async 3D fetch delay
- Alternatively, do this in `applySingleGeneratedBin` itself in the timeslicing mode store

**Files:** `src/components/dashboard-demo/DemoSlicePanel.tsx` or `src/store/useDashboardDemoTimeslicingModeStore.ts`

### Task 3: Make map respond to active slice

**Why:** The map shows all crimes in the viewport time range with no awareness of the active slice. Users cannot see which crime points belong to the current slice.

**What:**
- In `MapVisualization.tsx`, read `activeSliceIndex` from the coordination store
- In `DemoMapVisualization.tsx`, expose a slice-time-range overlay or pass the active slice info
- Add a visual indicator on the map showing the active slice's time-scoped crime points
- Approach: Import `useSliceDomainStore` and find the active slice by index from the coordination store, then cross-reference to get its time range. Filter `data` to only show points within that range when an active slice is set.
- Add a legend label like "Showing: [Burst 3] data" when filtered by active slice

**Files:** `src/components/map/MapVisualization.tsx`, `src/components/dashboard-demo/DemoMapVisualization.tsx`

### Task 4: Guard 3D view override of `activeSliceIndex` on data refetch

**Why:** `Demo3dSpatialView` line 189-191 sets `activeSliceIndex` to the last slice whenever `countedSlices.length` changes. This includes data refetches that happen due to store updates, which overrides user's current scrub position during playback or inspection.

**What:**
- Track whether the user has actively interacted with the active slice (e.g., via scrubber or playback)
- Use `lastInteractionAt` or a new flag to guard the override
- Only set to last slice if `activeSliceIndex` is 0 (the default) or if the count decreased (to avoid index OOB)
- Simplest approach: only override if `activeSliceIndex >= countedSlices.length` (OOB guard) or if `activeSliceIndex === 0` (initial state)

**Files:** `src/components/dashboard-demo/Demo3dSpatialView.tsx`

### Task 5 (stretch): Add reconciliation tracking for active slice changes

**Why:** The coordination store has `commitSelection`/`reconcileSelection` methods for tracking cross-panel sync status, but `activeSliceIndex` changes bypass them.

**What:**
- Update `setActiveSliceIndex` to also set `syncStatus` to `'syncing'`
- Add reconciliation checks in panels to call `reconcileSelection` when they confirm they've reflected the change
- Low priority — only if the basic coordination works but sync status reporting is needed

**Files:** `src/store/useDashboardDemoCoordinationStore.ts`

## Validation

1. **Manual test — apply flow**: Open Detect, generate bins, apply one. Verify that:
   - Inspect tab shows the applied slice as active
   - Timeline highlights the applied slice
   - Map shows crime points scoped to that slice's time range

2. **Manual test — playback**: Start playback in InspectPanel. Verify that:
   - 3D view cycles through slices
   - Timeline highlight shifts with each play step
   - Map updates to show each slice's data

3. **Manual test — scrubber**: Drag the SliceScrubber. Verify that:
   - 3D view updates immediately
   - Timeline highlight follows immediately
   - Map updates immediately

4. **Manual test — refetch stability**: While scrubbing, trigger a refetch (e.g., toggle visibility). Verify that the scrub position is not lost.
