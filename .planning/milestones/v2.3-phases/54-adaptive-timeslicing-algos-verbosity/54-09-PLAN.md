<objective>
Implement Option C for `/timeslicing-algos`: keep reference/context fetch stable, add a dedicated selection-specific fetch path for high-volume detail, and make dataset provenance explicit across detail rendering and diagnostics.

Purpose: QA currently cannot inspect dense selections beyond the context 50k fetch ceiling, and diagnostics/data-source provenance is ambiguous under sampling/fallback conditions. This plan introduces a second selection-focused data path with guardrails, preserves existing reference semantics, and makes data-source decisions visible.
Output: Route wiring for stable context + high-volume selection detail fetch, explicit sampled/full UX labels, diagnostics dataset-source control/provenance, guardrailed fallback behavior, and regression tests.
</objective>
