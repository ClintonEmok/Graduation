<objective>
Implement the default v3.0 generate-review-apply workflow for timeslicing.

Purpose: Let users describe their investigative intent with domain constraints, generate flexible bins without relying on a fixed bin count, review the generated slices clearly on the timeline, and apply them as the active slice set used by later coordinated views.

Output: A working `/timeslicing` flow where generated bins are the default starting point and apply is the obvious next step.
</objective>
