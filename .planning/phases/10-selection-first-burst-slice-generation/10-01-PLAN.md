<objective>
Redesign burst slice generation so the brushed selection is partitioned into hourly or daily base bins, scored, and expanded non-uniformly while preserving full coverage of the selected interval.

Purpose: replace the old density-window candidate model with a deterministic, brush-first time-slicing model that can visibly expand dense periods without dropping sparse ones.
Output: a tested helper contract for granularity-aware partitioning, burst scoring, and coverage invariants.
</objective>

<feature>
  <name>Granularity-aware non-uniform time slicing</name>
  <files>src/components/dashboard-demo/lib/demo-burst-generation.ts, src/components/dashboard-demo/lib/demo-burst-generation.test.ts</files>
  <behavior>
    - Given a brushed selection, partition it into uniform hourly or daily bins.
    - Score each bin with burstiness or equivalent event-spacing evidence.
    - Map score to warp weight / expansion while keeping every bin represented.
    - Return a neutral partition rather than an empty result when no bin stands out.
  </behavior>
  <implementation>
    Build a pure helper that accepts the brushed selection, selected granularity, and supporting timestamp inputs, then emits a draft-ready partition object with per-bin burst metadata and warp weights. The helper should guarantee exact coverage of the selection and keep the old overlap-error path out of the default demo flow.
  </implementation>
</feature>
