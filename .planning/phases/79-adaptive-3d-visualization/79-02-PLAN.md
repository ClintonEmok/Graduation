<task type="auto">
  <name>Task 2: Drag-to-resize slice boundaries in 3D</name>
  <files>src/app/stkde-3d/components/StkdeSliceStack.tsx</files>
  <action>
    Add drag handles to the selected slice for resizing its time range.

    When an applied slice is selected (isActive === true):
    1. Render two small sphere meshes as drag handles at the top and bottom of the slice volume:
       - Top handle Y = `sliceY + thickness / 2`
       - Bottom handle Y = `sliceY - thickness / 2`
       - Position X = 50 (right edge), Z = 0
       - Small visible spheres (radius ~0.8) with a bright color (white or cyan)
    2. onPointerDown on a handle → capture pointer (reuse pattern from SlicePlane.tsx)
    3. onPointerMove while dragging:
       - Raycast from pointer to get world Y position
       - Convert Y to epoch time using the inverse warp map (if adaptive) or linear mapping
       - For top handle: new endEpoch = computed time
       - For bottom handle: new startEpoch = computed time
       - Clamp so startEpoch &lt;= endEpoch with minimum duration
    4. onPointerUp → commit the range update:
        - Update the applied slice via `useSliceDomainStore`
        - Pending drafts are not rendered in the 3D view
    5. During drag, show a tooltip with the current date range (HTML overlay)

    Helper for Y-to-epoch mapping (inverse of sampleWarpSeconds):
    ```typescript
    const yToEpoch = (y: number): number => {
      // Map Y from scene coordinates back to normalized time
      const normalized = (y - START_Y) / (END_Y - START_Y);
      // If adaptive: invert through warp map (binary search)
      // If linear: normalized is already the linear position
      return domainStart + normalized * domainSpan;
    };
    ```

    The binary search for inverse warp can use the same approach as `toLinearSeconds` in `useScaleTransforms.ts`.
  </action>
