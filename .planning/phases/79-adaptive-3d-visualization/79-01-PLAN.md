  <files>src/app/stkde-3d/components/AdaptiveWarpAxis.tsx (new)</files>
  <action>
    Create a new React component that renders the adaptive warp map as 1024 volumetric boxes stacked vertically. Use instancedMesh with boxGeometry (following the pattern from AggregatedBars.tsx) for performance.

    Inputs (from useDashboardDemoCoordinationStore):
    - warpMap: Float32Array — cumulative display positions of bin boundaries
    - densityMap: Float32Array — per-bin density values [0, 1]
    - timeScaleMode: 'linear' | 'adaptive'
    - warpFactor: number (0 = linear, 1+ = fully adaptive)

    Rendering logic:
    1. Extract the first and last bin from the warp map to get startDisplay and endDisplay
     2. Compute totalDisplaySpan = warpMap[N-1] - warpMap[0] (or assume full height range)
     3. For each bin i (0 to N-1):
       - binDisplayStart = warpMap[i]
       - binDisplayEnd = (i &lt; N-1) ? warpMap[i+1] : warpMap[N-1]
       - binHeight = binDisplayEnd - binDisplayStart
       - normalizedHeight = (binHeight / totalDisplaySpan) * availableHeight
       - yPosition = cumulative sum of normalized heights (centered)
       - color = densityMap[i] mapped through a warm gradient:
         - 0.0 (sparse): cool blue (#1e3a5f)
         - 0.5 (moderate): teal (#0ea5e9)
         - 0.8 (dense): amber (#f59e0b)
         - 1.0 (max): red (#ef4444)
     4. When timeScaleMode === 'linear' or warpFactor === 0:
       - All bins equal height (100 / 1024 ≈ 0.098 units each)
       - Uniform low-opacity color
    5. Opacity: ~15% overall so slices remain visible through it
    6. Position: spans full XZ extent [-50, +50], stacked along Y
    7. renderOrder: behind StkdeSliceStack but above the map background

    Performance: use a single InstancedMesh with 1024 instances. Rebuild instance matrix and colors whenever warpMap changes.

    The component should be mountable as: &lt;AdaptiveWarpAxis /&gt;
  </action>

  <name>Task 3: Variable slice spacing from warp map</name>
  <files>src/app/stkde-3d/components/StkdeSliceStack.tsx</files>
  <action>
    Replace the fixed Y-position formula with warp-map-based positioning.

    Current: `const y = compact ? 0 : yForIndex(i);` where `yForIndex(i) = START_Y + i * SLICE_SPACING`

    New logic:
    1. Read `timeScaleMode`, `warpMap`, `warpFactor` from `useDashboardDemoCoordinationStore`
    2. If `timeScaleMode === 'adaptive'` and `warpMap` exists:
       - Use the current viewport time window as the domain (`viewportStart`, `viewportEnd`)
       - For each applied slice at index `i`, get its midpoint epoch time:
          `midEpoch = (slices[i].startEpoch + slices[i].endEpoch) / 2`
       - Normalize: `t = (midEpoch - viewportStart) / (viewportEnd - viewportStart)`
       - Sample the warp map at t: `sampleWarpSeconds(t, warpMap, [viewportStart, viewportEnd])`
       - Map the sampled display position to the Y range:
          `y = mapRange(displayPos, viewportStart, viewportEnd, START_Y, END_Y)`
    3. If `timeScaleMode === 'linear'` or no warp map: use the original fixed `yForIndex(i)`
    4. All dependent calculations (slab thickness, surfaceY, gridHelper, labels, ring effects) already use the `y` variable — they will automatically reposition.

    Helper function to add (reuse from useScaleTransforms.ts if possible, or inline):
    ```typescript
    const sampleWarpSeconds = (linearSec: number, warpMap: Float32Array, domain: [number, number]): number => {
      const [domainMin, domainMax] = domain;
      const span = Math.max(1e-9, domainMax - domainMin);
      const normalized = Math.max(0, Math.min(1, (linearSec - domainMin) / span));
      const idx = normalized * (warpMap.length - 1);
      const low = Math.floor(idx);
      const high = Math.min(low + 1, warpMap.length - 1);
      const frac = idx - low;
      return warpMap[low] * (1 - frac) + warpMap[high] * frac;
    };
    ```

    Note: The StkdeSliceStack receives applied slices through props (not from store directly). The slice objects have `startEpoch` and `endEpoch` as `EvolvingSlice` type fields. Verify the type has these fields.
  </action>
