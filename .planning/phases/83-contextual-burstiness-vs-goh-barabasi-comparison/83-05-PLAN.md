    Create `metrics/decision_gate.py` with two public functions.

    Module docstring: "Decision-gate evaluator (CBP-05) and DECISION-GATE.md writer (CBP-06). Reads comparison_table.csv, applies the locked thresholds (CV ratio >= 2x, range ratio >= 3x, absolute CV floor >= 0.10), and writes a verdict + justification to output/DECISION-GATE.md."

    1. Module-level constants (locked thresholds, no magic numbers):
       ```python
       PRIMARY_CV_RATIO = 2.0       # CONTEXT.md L42, L91
       PRIMARY_RANGE_RATIO = 3.0   # CONTEXT.md L42, L91
       SECONDARY_ABSOLUTE_CV_FLOOR = 0.10  # 83-RESEARCH.md L461-463 (NEW from research)
       TARGET_WINDOW_SEC = 86400    # the decision-gate target is the 1d window
       ```

    2. `def evaluate_gate(comparison_csv: Path) -> dict:`
       - Input: path to the comparison_table.csv.
       - Output: a dict with keys:
         - `verdict`: one of `"go"`, `"not_yet"`, `"no"`
         - `contextual_cv_1d`, `contextual_range_1d`, `gb_cv_1d`, `gb_range_1d`: float
         - `cv_ratio_1d`, `range_ratio_1d`: float (the contextual / gb ratio)
         - `primary_pass`: bool (primary rule result)
         - `secondary_pass`: bool (secondary rule result)
         - `rationale`: str (one-line summary of the verdict)
       - Algorithm:
         a. Load the CSV with `pd.read_csv(comparison_csv)`.
         b. Filter to `window_size == TARGET_WINDOW_SEC` and split into two rows: one with `metric == "contextual"`, one with `metric == "goh_barabasi"`.
         c. Extract `cv`, `range` from each.
         d. `cv_ratio_1d = contextual_cv / gb_cv if gb_cv > 0 else float("inf")`.
         e. `range_ratio_1d = contextual_range / gb_range if gb_range > 0 else float("inf")`.
         f. `primary_pass = (cv_ratio_1d >= PRIMARY_CV_RATIO) and (range_ratio_1d >= PRIMARY_RANGE_RATIO)`.
         g. `secondary_pass = contextual_cv_1d >= SECONDARY_ABSOLUTE_CV_FLOOR`.
         h. Verdict logic:
            - `"go"` if `primary_pass and secondary_pass`
            - `"not_yet"` if `primary_pass or secondary_pass` (one passes, one doesn't — borderline)
            - `"no"` if `not primary_pass and not secondary_pass` (both fail)
         i. `rationale = f"CV ratio {cv_ratio_1d:.2f}x (need >= 2.00x), range ratio {range_ratio_1d:.2f}x (need >= 3.00x), absolute CV {contextual_cv_1d:.3f} (need >= 0.10)"`.
         j. Return the dict.

    3. `def write_decision_gate_md(evaluation: dict, output_path: Path, *, source_csv: Path | None = None) -> None:`
       - Input: the dict from `evaluate_gate`, the path to write to, and (optionally) the source CSV path for traceability.
       - Output: writes the markdown to `output_path`.
       - Markdown structure (use exactly this format — the orchestrator may grep for these headings):
         ```markdown
         # Phase 83 Decision Gate: Contextual Burstiness vs Goh-Barabasi

         **Date:** 2026-06-27
         **Source data:** 8.5M-record Chicago crime dataset (2001-2026)
         **Decision window:** 1d (86400s)

         ## Verdict: **{verdict_upper}**

         {rationale}

         ## Per-metric summary at 1d

         | Metric | CV | Range | n_windows |
         |---|---:|---:|---:|
         | Contextual (z) | {contextual_cv_1d:.4f} | {contextual_range_1d:.4f} | (see comparison_table.csv) |
         | Goh-Barabasi (B) | {gb_cv_1d:.4f} | {gb_range_1d:.4f} | (see comparison_table.csv) |
         | **Ratio (contextual / gb)** | **{cv_ratio_1d:.2f}x** | **{range_ratio_1d:.2f}x** | — |

         ## Threshold rationale

         The 2x CV and 3x range rules guard against: (a) the contextual metric being so noisy that its CV advantage is meaningless, and (b) the contextual metric being so tightly bounded that it can't drive a warp. The 2x threshold is conservative — it is the minimum ratio at which a thesis can credibly claim the new metric is a *material* improvement over the old.

         The absolute CV floor of 0.10 (the secondary criterion, sourced from 83-RESEARCH.md L461-463) is added to handle the documented near-zero-denominator case: Goh-Barabasi B is bounded in [-1, 1] and the prior 100k-sample baseline (docs/FUTURE-WORK-ADAPTIVE-TIME.md:288) shows B CV at 1d = 0.01. A 2x ratio against a denominator of 0.005 is technically a 2x advantage but is still essentially "no signal" — the absolute floor guards against this false positive.

         ## Next steps (CBP-06)

         Based on the verdict **{verdict_lower}**:

         - If **"go"**: Phase 84 (Burstiness Signal Contract + Density Fallback, BFT-01..BFT-12) is **unlocked**. Begin /gsd-plan-phase 84.
         - If **"not_yet"**: The 2,016-cell stretch baseline (hour × dayOfWeek × month, per 83-RESEARCH.md L147-149 + L489) is the recommended next move. Re-run this analysis with the stretch baseline and re-evaluate the gate. If it still fails, fall back to BFT-04..BFT-06 in Phase 84 (density-driven detail timeline) and close the contextual-burstiness integration thread.
         - If **"no"**: Close the contextual-burstiness prototype-integration thread. Fall back to the density-driven detail timeline (BFT-04..BFT-06) in Phase 84 and document the negative result in the thesis discussion.

         ## Artifacts

         - `output/contextual_metric.parquet` — per-window z-scores
         - `output/goh_barabasi_metric.parquet` — per-window B values
         - `output/comparison_table.csv` — per-metric CV/range summary
         - `output/figures/z_heatmap.png` — 24×7 z-score heatmap
         - `output/figures/per_window_timeseries.png` — twin-axis B vs z
         - `output/figures/contrast_table.png` — visual contrast table
         - `output/DECISION-GATE.md` — this document
         ```
       - `verdict_upper` is the verdict title-cased (e.g. "GO"); `verdict_lower` is lowercase.

       - Print `f"Wrote decision gate ({evaluation['verdict']}) to {output_path}"`.
