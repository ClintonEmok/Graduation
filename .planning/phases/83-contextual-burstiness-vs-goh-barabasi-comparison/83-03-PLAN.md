<objective>
Implement the three **reference metrics** (B, density, CV) over the same 1h/6h/1d/1w window sweep used for the contextual metric. Each is a per-window summary statistic operating on the same timestamps. The outputs are three parquet files Plan 04 will compare against the contextual z-score on a 4-way CV/range table.

**Reference metrics:**
- **B** (Goh-Barabasi, CBP-02): `(σ_τ − μ_τ) / (σ_τ + μ_τ)` on inter-event times τ — direct port of `src/lib/burst-detection.ts:53-59`
- **density** (CBP-07): `n_events / window_seconds` — events per second, the simplest possible "burst" signal
- **CV** (CBP-08): `σ_τ / μ_τ` on inter-event times τ — the unbounded cousin of B, isolates the effect of B's bounded denominator

Purpose: CBP-02, CBP-07, CBP-08 require three reference metrics over the same windows as the contextual metric so the comparison in CBP-03 can be 4-way (contextual, B, density, CV). This plan delivers the three reference metrics + sweeps + parquets plus a sanity check for B against the documented 1h baseline (≈0.30 from `docs/FUTURE-WORK-ADAPTIVE-TIME.md:288`).

Output: `metrics/goh_barabasi.py` (B), `metrics/density.py` (density), `metrics/cv.py` (CV); unit tests for all three; and `output/{goh_barabasi_metric,density_metric,cv_metric}.parquet`.
</objective>

            - Append `(ws, we, n_events, b)`.
