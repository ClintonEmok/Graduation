# Phase 60 Plan 1: Integration Milestone

## Objective

Integrate all separate parts of the neon-tiger crime analysis application into a cohesive system with user-driven timeslicing, dynamic binning, proper map visualization, and functional dashboard.

## Context Files

- @src/utils/binning.ts - Current fixed tick binning
- @src/store/useAdaptiveStore.ts - Current adaptive timeslicing store
- @src/app/dashboard/page.tsx - Current dashboard layout
- @src/components/map/MapVisualization.tsx - Current map component
- @src/workers/stkdeHotspot.worker.ts - STKDE worker

## Tasks

### Task 1: Dynamic Binning System with Rules (5/5)

Create a rule-based dynamic binning system that replaces fixed tick count with constraints-driven bin generation.

**Implementation:**

1. Create `src/lib/binning/rules.ts` - Rule definitions for bin strategies:
   - `BinningRule` interface with conditions and parameters
   - Preset rules: "daytime-heavy", "nighttime-heavy", "crime-type-specific", "burstiness", "uniform-distribution"
   - Custom rule creation

2. Update `src/utils/binning.ts` to accept rules:
   - Replace `ticks: number` with `BinningConfig` object
   - Implement rule-based threshold calculation
   - Support dynamic bin count (not fixed)

3. Create `src/components/binning/BinningControls.tsx` - UI for:
   - Selecting binning strategy
   - Merging adjacent bins
   - Splitting bins
   - Deleting bins
   - Resizing bins (adjusting time ranges)
   - Saving/loading bin configurations

4. Add bin constraint validation:
   - Ensure minimum events per bin
   - Ensure minimum time span per bin
   - Prevent overlapping bins
   - Validate bin continuity

5. Create `src/store/useBinningStore.ts`:
   - Track current binning rules and configuration
   - Store user-modified bins
   - Persist bin configurations

**Done Criteria:**
- [x] Rule-based binning replaces fixed tick count
- [x] UI allows merge/split/delete/resize operations
- [x] Bins update in real-time based on selected rule
- [x] Configuration can be saved/loaded

### Task 2: User-Driven Timeslicing (5/5)

Transform timeslicing from automatic computation to user-driven interactive controls.

**Implementation:**

1. Enhance `src/store/useAdaptiveStore.ts`:
   - Add `isManualMode: boolean` to allow user override
   - Add manual slice creation endpoints
   - Add slice boundary editing

2. Create `src/components/timeslicing/ManualTimesliceEditor.tsx`:
   - Drag handles to create/adjust slice boundaries
   - Visual feedback on slice boundaries
   - Snap to significant time points (hour/day/week boundaries)

3. Create `src/components/timeslicing/TimesliceToolbar.tsx`:
   - Toggle between auto and manual mode
   - Preset time slice options
   - Clear all slices button
   - Import/export slice configurations

4. Create `src/hooks/useSliceInteraction.ts`:
   - Handle slice creation via click-drag
   - Handle slice boundary adjustment
   - Handle multi-slice selection for batch operations
   - Keyboard shortcuts for common operations

5. Integrate with existing slice stores:
   - Connect manual editor to `useSliceStore`
   - Connect to `useSliceSelectionStore`
   - Connect to `useSliceAdjustmentStore`

**Done Criteria:**
- [x] User can toggle between automatic and manual timeslicing
- [x] Manual mode allows click-drag slice creation
- [x] Slice boundaries can be adjusted by dragging
- [x] Existing auto-computation still works as fallback
