<objective>
Introduce the parameterized burstiness/density/contextual signal contract on the dashboard-demo adaptive timeline. Define `AdaptiveSignalSource` and the mapper interface, add `activeSignalSource` to `useAdaptiveStore` (default `'burstiness'`, persisted in localStorage), and refactor `addSliceFromBin` / `replaceSlicesFromBins` to dispatch on the source. The burstiness path must reproduce pre-Phase-84 `warpWeight` values (`1.0` for non-burst, `bin.warpWeight ?? 1.25` for burst taxonomy) exactly — a parity test guards against drift. Density and contextual ship as stubs that return `1.0` when no baseline is loaded; their real implementations land in 84-02 and 84-03 respectively.

Purpose: BFT-01 requires a parameterized signal contract; BFT-02 requires burstiness as the default with density/contextual as fallbacks; BFT-03 preserves the density path. BFT-12 requires the public timeline API to survive source switching. The parity test is the load-bearing assertion that the refactor does not regress pre-Phase-84 behavior.

Output: A new `src/lib/signal-sources/` module with the type contract + three mappers; an `activeSignalSource` field in `useAdaptiveStore` with persist middleware; refactored `addSliceFromBin` / `replaceSlicesFromBins`; parity and guard tests.
</objective>
