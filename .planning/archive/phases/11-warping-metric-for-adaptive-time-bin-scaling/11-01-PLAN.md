<objective>
Create a pure warp-scoring helper for comparable bins so later consumers can expand or compress widths without reordering or collapsing the selection.

Purpose: centralize the score math before UI wiring so the shared contract is testable and reusable.
Output: a tested bin-scaling helper contract plus the edge cases that lock order and minimum-width behavior.
</objective>

<feature>
  <name>Same-granularity warp scoring</name>
  <files>src/lib/binning/warp-scaling.ts, src/lib/binning/warp-scaling.test.ts</files>
  <behavior>
    - Given bins from one granularity, compute relative scores against peer bins only.
    - Convert scores into warp weights or width factors while preserving input order.
    - Clamp the minimum effective width above zero.
    - Return a neutral fallback for empty, invalid, or fully flat input.
  </behavior>
  <implementation>
    Build a pure helper that accepts a comparable-bin adapter shape (id, startTime, endTime, count, granularity, and optional hint weight) and returns scored bins plus a monotonic warp map. Use deterministic tie handling so equal bins stay in input order and the helper can be consumed by both the showcase route and authored warp-map preview.
  </implementation>
</feature>
